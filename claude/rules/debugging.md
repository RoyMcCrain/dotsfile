# Debugging Rule

## 基本方針

修正前に原因を特定する。推測で直さない。

## 必須手順

1. エラーを再現する
2. 原因をコード / 環境・デプロイ / 設定の観点で切り分ける
3. 確度の高い仮説から検証する
4. 原因が確定してから修正する

## 禁止事項

- 再現せずに修正を始める
- 同じアプローチを2回以上繰り返す（失敗したら別角度で）
- 正常動作をバグと判断して修正する（ユーザーに確認）
- 修正が不要と判明したら即座に作業を中止する

## 環境起因を疑うサイン

- 401 / 403 / 404 エラー
- ローカルで動くがデプロイ先で動かない
- 最近デプロイした直後の問題
