# Title[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#title "Permanent link")

> A cool name for your Project

Author: [Your-Name](mailto:your-name@reachable.com)

> If there are multiple authors, just list them all

## Summary[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#summary "Permanent link")

A short summary of your project/re-design/component and what problems it
addresses in about 3-10 sentences.

## State of the Feature as of `$VERSION` (optional)[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#state-of-the-feature-as-of-version-optional "Permanent link")

The state of the feature you want to improve and where it currently falls
short. If there's nothing to compare to, leave it out.

## Prior work (optional)[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#prior-work-optional "Permanent link")

Does this feature exist somewhere else and which tradeoffs it made.

> If there's no prior work, then use the related work section below.

## Goals and non-goals[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#goals-and-non-goals "Permanent link")

Direct goals of the project and features deemed not worth pursuing.

## Overview[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#overview "Permanent link")

A detailed overview of the project and the improvements it brings.

### Detailed Design[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#detailed-design "Permanent link")

The place to describe all new interfaces and interactions and how it plays into
the existing code and behavior. This is the place for all nitty-gritty details
which interact with the system.

## Alternatives considered (optional)[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#alternatives-considered-optional "Permanent link")

Other alternatives to your suggested approach, and why they fall short.

## Issues addressed (optional)[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#issues-addressed-optional "Permanent link")

A list of issues which are addressed by this design.

## Related Work (optional)[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#related-work-optional "Permanent link")

If there's a feature in another VCS which shares some similarities to your
proposed work, it belongs here. An example would be Jujutsu sparse workspaces
and Perforce client workspaces.

## Future Possibilities[¶](https://www.jj-vcs.dev/latest/design_doc_blueprint/index.html#future-possibilities "Permanent link")

The section for things which could be added to it or deemed out of scope during
the discussion.
